CPK*

Easy Plugin Development for Pentaho

*Community Plugin Kickstarter

Motivation

Pentaho is very well known for being a very good Business Analytics software, but is in fact much more than that; Pentaho is a great platform to build on top of.

(Totally stolen from Kaapa's blog)

Goal

The goal of CPK is to provide a simple and easy way to develop pentaho plugins that behave like packaged applications, simplifying its structure.

(Aaaand again :P )

Plugin structure

UI

Accessing Endpoints and Dashboards

Endpoints are listed from the /endpoints folder. Dashboards from the /dashboards folder.

They can be accessed using:

/pentaho/content/pluginName/endpointName

Passing parameters to Kettle


Endpoint:

paramcustomParam

Kettle:

${customParam}
*

.../pluginName/endpointName?paramcustomParam=true

* Same mapping as CDA

Other Features and Roadmap

Access Control*

Non admin users can not access dashboards, or endpoints, inside the /admin folder, and the sitemap does not show them.

*(Based on Pentaho roles)

Not admin

cvb/filelist?userid=pat&password=password


Admin

cvb/filelist?userid=joe&password=password

Download Zip*

Although CVB does not expose that at the moment, folders can be downloaded in a .zip file.

CPK triggers zip when the kettle result is more than one file.
